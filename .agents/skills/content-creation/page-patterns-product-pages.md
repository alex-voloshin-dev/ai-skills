# Page Patterns: Product and Marketing Pages

## Landing Page

Recommended order:

1. hero
2. social proof bar
3. problem statement
4. solution or how it works
5. feature-benefit sections
6. detailed proof
7. pricing or comparison
8. FAQ
9. final CTA
10. footer

Section guidelines:

- hero should pass the 5-second clarity test
- testimonials should include concrete results
- final CTA should change wording but repeat the same core promise

## SaaS Homepage

Recommended order:

1. hero
2. logo bar
3. value pillars
4. product showcase
5. integrations
6. testimonials
7. security or compliance
8. CTA section

Use real product visuals where possible and keep differentiation explicit.

## Feature Page

Recommended order:

1. feature hero
2. problem -> solution framing
3. capability details
4. use cases
5. optional comparison
6. feature-specific testimonial
7. related features
8. feature CTA

Organize sub-features by the user workflow, not by internal implementation order.

## About Page

Recommended order:

1. mission hero
2. story
3. values
4. team
5. numbers
6. partners or investors
7. CTA aligned to likely visitor intent

Keep the story specific and evidence-backed. Generic mission prose weakens trust.

## Pricing Page

Recommended order:

1. pricing header
2. pricing tiers
3. comparison table
4. FAQ
5. social proof
6. enterprise path
7. guarantee or risk reversal

Pricing guidance:

- make upgrade paths obvious
- keep CTA language aligned to buyer intent
- answer objections close to the pricing table
