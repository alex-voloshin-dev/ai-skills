# Content Tools Guide

Use this file as the overview for content workflows and AI-assisted production. Detailed operating guides now live in narrower companion files.

## Workflow Map

| Need | File |
|---|---|
| Topic research, fact gathering, internal cross-link planning, blog briefing | `content-research-workflows.md` |
| AI text generation, image generation, and prompt templates | `ai-content-generation-workflows.md` |
| Content ops, service integration, publishing flow, and measurement | `content-ops-and-measurement.md` |

## Recommended Sequence

1. research first
2. create the content brief
3. generate draft options
4. review for brand, accuracy, legal, accessibility, and SEO
5. publish with discovery and analytics in place
6. iterate from measured performance

## Guardrails

- do not publish unverifiable claims
- keep tool choice proportional to the task
- treat AI-generated visuals like any other asset: review quality, license, and bias
- humanize all public-facing copy before finalization
