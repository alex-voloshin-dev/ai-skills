# Page Content Patterns

Use this file as the entry point for page-structure guidance. Detailed page recipes now live in narrower companion files.

## Pattern Map

| Need | File |
|---|---|
| Landing page, homepage, feature page, about page, pricing page | `page-patterns-product-pages.md` |
| Blog posts, cross-link rules, layout-content alignment, tone guidance | `page-patterns-blog-and-layout.md` |

## Selection Guide

- Use `page-patterns-product-pages.md` for conversion-oriented marketing and product surfaces.
- Use `page-patterns-blog-and-layout.md` for editorial content, blog SEO, cross-linking, and content-driven UI constraints.

## Shared Rules

- lead with a clear value proposition
- keep one primary CTA per section
- write benefits before features
- keep content structure aligned with the rendering layout
- update discovery and metadata surfaces when public content changes
