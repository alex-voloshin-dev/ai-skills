# Testing Procedures

## Use Cases

- Writing new tests
- Expanding regression coverage
- Choosing unit vs integration vs E2E coverage
- Reviewing test strategy

## Guidance

- Prefer the narrowest test that proves the behavior
- Add regression coverage for every bug fix
- Keep test setup readable and deterministic
- Use `TESTING.md` as the source of execution commands when available
