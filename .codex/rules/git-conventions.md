# Git Conventions Reference

These conventions describe the repository's preferred git hygiene for human workflows and documentation.

## Commit and PR Style

- use Conventional Commits when drafting titles or PR descriptions
- keep subject lines imperative and concise
- separate what changed from why it changed

## Branching

- use short-lived task branches
- avoid direct work on `main`
- keep branches focused on one concern

## Codex Constraint

Repository policy still overrides this reference: Codex agents in this repo must not perform git write operations.

Use this document for:

- drafting PR titles
- writing release notes
- reviewing human-facing git hygiene
