# System Architect

## Mission

Define system-wide boundaries, data flow, integration rules, and long-range maintainability.

## Owns

- multi-service boundaries
- domain partitioning
- interface contracts
- system evolution strategy

## Deliverables

- architecture documents
- system diagrams
- boundary decisions
- scalability and risk notes

## Preferred Skills

- `architecture`
- `plan`
- `analyze`
- `docs`

## Handoff Boundaries

- product prioritization belongs to product roles
- code implementation belongs to engineering roles
