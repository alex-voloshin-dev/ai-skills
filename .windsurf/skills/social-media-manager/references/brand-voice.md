# Brand Voice Guide

Source of truth: `marketing/MARKETING.md` in the project root.
Read it before writing any post. It contains: product definition, ICP,
positioning, team members, and vocabulary.

This file provides generic tone and structure guidance that applies
regardless of the specific product.

## Required context from MARKETING.md

Before writing, extract these from `marketing/MARKETING.md`:

- **Product name** and one-sentence description
- **Key differentiators** (what makes it unique vs competitors)
- **Target audience** (primary, secondary)
- **Domain** (for plain-text CTAs)
- **Team members** who will post from personal accounts
- **Vocabulary** (terms to use and terms to avoid)
- **Boilerplates** (short and medium product descriptions)

If any of these are missing from MARKETING.md, ask the user before writing.

## Key conversion points to weave into posts

- Highlight the lowest-friction CTA available (free tier, no signup, quick start)
- Emphasize low time commitment ("takes 5 minutes", "30 seconds")
- Mention the domain as plain text, never as a hyperlink (algorithm penalty)

## Personality

Technical, direct, evidence-based, approachable, no-BS.

Adapt personality to the product's positioning in MARKETING.md. If the product
targets developers, lean technical. If it targets marketers, lean practical.
If it targets consumers, lean conversational.

## Tone by platform

- **X/Twitter**: Conversational but factual. Short sentences. Data when possible.
- **LinkedIn**: Professional peer sharing an observation. Not a whitepaper.
- **Facebook**: Casual, relatable. Shorter than LinkedIn.

## Vocabulary

### Always avoid (regardless of product)

Never use: "revolutionary", "game-changing", "best-in-class", "synergy",
"leverage" (as verb), "disrupt", "groundbreaking", "cutting-edge",
"delve", "crucial", "landscape", "pivotal", "tapestry", "vibrant".

### Product-specific vocabulary

Read from the vocabulary section of `marketing/MARKETING.md`.
Use the product's own terminology consistently. Don't invent synonyms
for established product terms.

## Boilerplates

Use the boilerplates defined in `marketing/MARKETING.md`. Every product
should have:
- **Short (25 words)**: one-sentence product description
- **Medium (50 words)**: expanded description with key features

If boilerplates don't exist in MARKETING.md, draft them and suggest
adding them to the strategy document.

## Objection responses (for comment replies)

Check `marketing/MARKETING.md` for product-specific objection responses.
If none exist, draft responses based on the product's positioning and
competitive landscape. Common objection categories:

| Category | How to respond |
|----------|---------------|
| "I can do this manually" | Emphasize scale, consistency, and time savings |
| "Tool X already does this" | Focus on the specific differentiator, not feature lists |
| "Too new / unproven" | Point to specific data, evidence, or early adopter results |
| "How accurate / reliable?" | Reference verification methods, transparency, evidence |
