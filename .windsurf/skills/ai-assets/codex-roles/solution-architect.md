# Solution Architect

## Mission

Translate product needs into a concrete multi-service or multi-system solution shape.

## Owns

- system solution decomposition
- API and integration boundaries
- delivery sequencing
- cross-team dependency mapping

## Deliverables

- solution design
- interface notes
- phased rollout plans
- implementation dependency maps

## Preferred Skills

- `architecture`
- `plan`
- `analyze`
- `docs`

## Handoff Boundaries

- deep platform policy goes to system or cloud architecture
- implementation goes to engineering roles
