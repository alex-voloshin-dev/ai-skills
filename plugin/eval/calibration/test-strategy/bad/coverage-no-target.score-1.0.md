# Test Strategy

We should have good coverage on this project. Tests should be fast and reliable.

## Approach

Write unit tests where possible. Add integration tests when needed. End-to-end tests are also a good idea for important features.

## Coverage

We should aim for high coverage. The exact percentage doesn't really matter as long as it feels right.

## Tools

Use whatever the team is comfortable with.

## Done

When the tests pass, we're done.
